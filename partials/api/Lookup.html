Lookup

Description

Allows for easier distinction when debugging if object type is shown as either Lookup or User. Also allows us to create an async request for the entity being referenced by the lookup

Usage

Lookup(s, options);

Parameters

ParamTypeDetails
sstring

String to split into lookupValue and lookupId

optionsobject

Contains a reference to the parent list item and the property name.

options.entityobject

Reference to parent list item.

options.propertyNameobject

Key on list item object.

Methods

  • getEntity()

    Allows us to create a promise that will resolve with the entity referenced in the lookup whenever that list item is registered.

    Returns
    promise

    Resolves with the object the lookup is referencing.

    Example

    var project = {
   title: 'Project 1',
   location: {
       lookupId: 5,
       lookupValue: 'Some Building'
   }
};

//To get the location entity
project.location.getEntity().then(function(entity) {
    //Resolves with the full location entity once it's registered in the cache

   });

  • getProperty(propertyPath)

    Returns a promise which resolves with the value of a property in the referenced object.

    Parameters
    ParamTypeDetails
    propertyPathstring

    The dot separated propertyPath.

    Returns
    promise

    Resolves with the value, or undefined if it doesn't exists.

    Example

    var project = {
   title: 'Project 1',
   location: {
       lookupId: 5,
       lookupValue: 'Some Building'
   }
};

//To get the location.city
project.location.getProperty('city').then(function(city) {
   //Resolves with the city property from the referenced location entity

   });
